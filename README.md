#优缺点总结
读取内容，使用一个或多个极短的缺点标签，如过热，续航差等。只需要标注出缺点且标签要求简短。若无缺点则写无，若有多个标签使用 | 进行分隔

读取内容，使用一个或多个极短的优点标签，如续航好等。只需要标注出优点且标签要求简短。若无优点则写无,若有多个标签使用 | 进行分隔

#相似子类型统一
读取表格中的缺点和优点，将有相似的缺点或者优点修改成行统一词语。
要求：
1.保留修改前的缺点或者优点与完整的原评论。
2.缺点或优点可以使用多个词语，未修改的优缺点保留到修改后的优缺点中
3.不做其他分析。
输出的报告要求：
第一列序号，第二列星级，第三列中文标题，第四列完整的中文原评论，第五列修改缺点，第六列修改后缺点，第七列修改前优点，第八列修改后优点

#大类分类
读取表格中的缺点和优点，将缺点和优点分别进行归类。
要求
1.每一个评论可以有一个或者多个类型。
2.将原来的缺点和优点作为对应类型的子类。
3.每一个子类只能对应一个与他相关的类型，输出分类结果时按照"类型（子类）"格式输出。
4.不做其他分析。
5.如果优缺点为无，则保留无
输出的报告要求：
第一列序号，第二列星级，第三列中文标题，第四列完整的中文原评论，第五列缺点分类结果，第六列优点分类结果

#VOC报告
